01 — Verification

One call. Full compliance check.

Wrap every agent payment in a single verify() call. OFAC screening, threshold checks, and compliance status returned in one response. The transaction is logged to your tamper-evident digest chain automatically.

  • OFAC sanctions screening on every address
  • Travel Rule ($3K), CTR ($10K), and large transaction ($50K) thresholds
  • Compliance status, risk level, and recommendations in the response
  • Transaction automatically logged to digest chain with full context
agent.ts 
const result = await ctx.verify({
  txHash: '0xabc...def',
  chain:  'base',
  amount: '5000',
  token:  'USDC',
  from:   '0xAgentWallet',
  to:     '0xRecipient',
  agentId: 'payment-agent-v2',
});

// result.compliant        → true
// result.riskLevel        → 'low'
// result.checks           → [{name, passed}]
// result.digestProof.valid → true

02 — Compliance Logging

Tamper-evident records. Certificates on demand.

Every agent action is captured in a SHA-256 linked digest chain. Each record includes the previous hash — altering one entry breaks every link after it. Generate compliance certificates with cryptographic proof at any time.

  • SHA-256 digest chain — each record links to the previous hash
  • Full context: agent ID, reasoning, metadata, timestamps
  • Compliance certificates with digest proof and trust score
  • Export as JSON or CSV for auditors and regulators
certificate.ts 
const cert = await ctx.generateComplianceCertificate({
  agentId: 'treasury-agent-v2',
  timeRange: { from: startOfMonth, to: now },
  includeReasoning: true,
});

// cert.complianceStatus   → 'compliant'
// cert.digestChain.verified → true
// cert.contentHash        → SHA-256
// cert.trustScore.score   → 92

const valid = ctx.verifyDigestChain();
// valid.intact → true (no tampering)

04 — OFAC Screening

Built-in SDN list. Pluggable providers.

Every transaction is screened against the Treasury Department's SDN list out of the box — no API key required. Need Chainalysis or OpenSanctions? Plug them in with one line. Results are logged to your audit trail automatically.

  • Built-in Treasury SDN list screening — zero config
  • Pluggable ScreeningProvider architecture
  • Chainalysis, OpenSanctions, or your own provider
  • Multi-provider aggregation with configurable consensus
  • Webhook and SMTP alerts on flagged addresses
screening.ts 
// Built-in — no API key needed
const check = UsdcCompliance.checkTransaction({
  from: '0xSender',
  to:   '0xRecipient',
  amount: '25000',
  chain: 'base',
});

// Or plug in Chainalysis:
const aggregator = new ScreeningAggregator({
  providers: [
    new TreasurySDNProvider(),
    new ChainalysisOracleProvider({
      apiKey: process.env.CHAINALYSIS_KEY
    }),
  ],
});

05 — Anomaly Detection

Six patterns. Real-time alerts.

Rule-based detection across six behavioral patterns. Configure thresholds per agent, get callbacks when anomalies fire, and route alerts to your compliance team via webhooks.

  • Unusual amounts — transactions outside normal range
  • Frequency spikes — sudden increase in transaction volume
  • New destinations — payments to previously unseen addresses
  • Off-hours activity — transactions outside business hours
  • Rapid succession — multiple transfers in short windows
  • Round amounts — suspiciously round payment values

06 — Trust Scoring

Real-time 0–100 score. Five weighted factors.

Every agent gets a trust score computed from its history, transaction patterns, and context. Use it to gate high-value operations, set dynamic approval thresholds, or feed into your own risk models.

  • Agent history — track record across all logged actions
  • Amount normality — deviation from typical transaction sizes
  • Transaction velocity — frequency relative to baseline
  • Recipient trust — history with destination addresses
  • Contextual patterns — time of day, chain, token consistency
